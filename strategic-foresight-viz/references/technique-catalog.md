# Strategic Foresight Technique Catalog

Based on course notes (I-Intelligence, July 2021) Pallaris et. al., and personal research.

## Informal upstream credits

The 24-technique catalog below is structured around the I-Intelligence course material, but most of the techniques themselves predate it. Best-effort attributions (not exhaustive, not formally cited):

- **PESTLE** — variant of ETPS, Francis J. Aguilar, *Scanning the Business Environment* (1967).
- **Macro / Meso / Micro Analysis** — general social-science framing, no single canonical author.
- **Trend Impact Analysis** — Theodore Gordon at RAND / The Futures Group (1970s).
- **Trend-on-Trend Cross-Impact** — Gordon & Hayward, cross-impact analysis (1968).
- **Perception Analysis (See / Hear / Think / Feel)** — empathy-mapping lineage, popularized by XPLANE (Dave Gray).
- **What / So What / Now What** — Driscoll's reflective model (1994) via Borton (1970).
- **Weak Signal Analysis** — Igor Ansoff, *Managing Strategic Surprise by Response to Weak Signals* (1975).
- **Three Futures (Best / Likely / Worst)** — common foresight shorthand, no single origin.
- **Futures Cone** — Trevor Hancock & Clement Bezold (1994), popularized by Joseph Voros (2003).
- **Dator's Four Archetypes** — Jim Dator, University of Hawaii (1979).
- **Intuitive Logics** — Pierre Wack at Shell (1970s); Peter Schwartz, *The Art of the Long View* (1991).
- **Lum's Four Steps to the Future / Layered Timeline / Stakeholder Analysis** — Richard A. K. Lum, *4 Steps to the Future* (2016).
- **VUCA** — US Army War College, late 1980s.
- **CATUR** — Pallaris extension of VUCA (I-Intelligence).
- **What If / Then Cross-Impact** — Tetlock-style counterfactual reasoning combined with Gordon cross-impact.
- **Consequence Analysis** — generic foresight framing, common in policy planning.
- **Analogical Reasoning** — Ernest May, *Lessons of the Past* (1973); Neustadt & May, *Thinking in Time* (1986).

Anything not listed above is based on course notes and personal research.

## Category 1: Environmental Scanning

### PESTLE/GRNO Scan
- **Category:** Environmental Scanning
- **Diagram type:** Multi-Column Matrix
- **Tool:** XML
- **Layout:** Two integrated matrices. Primary matrix: 6 columns (Political, Economic, Socio-Cultural, Technological, Legal, Environmental) × 2 rows (Trends, Events). Secondary matrix: same 6 PESTLE columns × 4 rows (Local, National, Regional/Europe, Global) showing geographic range overlay.
- **User inquiry fields:**
  1. Topic or domain being scanned (e.g., "Healthcare technology in Europe")
  2. Geographic scope focus (specify if primary focus is local, national, regional, or global)
  3. At least one trend for each PESTLE dimension (Political, Economic, Socio-Cultural, Technological, Legal, Environmental)
  4. At least one event for each PESTLE dimension
  5. Geographic range assessment (how each trend/event applies at local, national, regional, and global levels)
- **Color mapping:** PESTLE columns use distinct colors per dimension; geographic rows use intensity gradient (pale = local, dark = global)

### Macro/Meso/Micro Analysis
- **Category:** Environmental Scanning
- **Diagram type:** Three-Row Layered Matrix
- **Tool:** XML
- **Layout:** 3 horizontal rows (Macro, Meso, Micro) × 6 columns representing user-defined factors. Each cell contains relevant observations at that analytical level.
- **User inquiry fields:**
  1. Domain or issue being analyzed (e.g., "Digital transformation in manufacturing")
  2. Six factor headers to structure the analysis (e.g., "Technology," "Regulation," "Consumer Behavior," "Competition," "Supply Chain," "Workforce")
  3. Macro-level observations (industry/societal scale factors)
  4. Meso-level observations (organizational/sectoral scale factors)
  5. Micro-level observations (individual/team/local scale factors)
- **Color mapping:** Rows use grayscale gradient (lightest = Macro/broadest, darkest = Micro/most specific)

## Category 2: Trend Analysis

### Trend Stage Analysis
- **Category:** Trend Analysis
- **Diagram type:** Quadrant/2×2 Matrix
- **Tool:** XML
- **Layout:** X-axis represents Lifecycle Stage (Latent → Emerging → Current → Established); Y-axis represents Impact (Low → Medium → High). Trends plotted as positioned nodes within the matrix.
- **User inquiry fields:**
  1. List of trends to analyze (3–10 trends recommended)
  2. For each trend: lifecycle stage classification (Latent, Emerging, Current, or Established)
  3. For each trend: impact level assessment (Low, Medium, or High)
  4. Brief description of each trend (optional, for labeling nodes)
- **Color mapping:** Quadrants colored by strategic priority: upper-right (high-impact, established) = red/critical; upper-left (high-impact, emerging) = orange/watch; lower-right (low-impact, established) = green/stable; lower-left (low-impact, latent) = blue/monitor

### Trend Impact Analysis (Layered)
- **Category:** Trend Analysis
- **Diagram type:** Comparison Table
- **Tool:** XML
- **Layout:** Rows represent individual trends (1–5); columns represent impact layers: Personal Impact, Organisational Impact, Country Impact, European Impact. Each cell contains severity or descriptive assessment.
- **User inquiry fields:**
  1. 3–5 identified trends
  2. For each trend: personal impact assessment (what effect on individuals/households)
  3. For each trend: organisational impact assessment (what effect on businesses/institutions)
  4. For each trend: country-level impact assessment (what effect on national scale)
  5. For each trend: European-level impact assessment (what effect on regional scale)
- **Color mapping:** Cells use intensity gradient based on impact severity (light green = low, yellow = medium, red = high impact)

### Trend Impact Analysis (Temporal)
- **Category:** Trend Analysis
- **Diagram type:** Comparison Table
- **Tool:** XML
- **Layout:** Rows represent individual trends (1–5); columns represent time horizons: 1 Year, 3 Years, 5 Years, 10 Years. Each cell shows projected impact at that time point.
- **User inquiry fields:**
  1. 3–5 identified trends
  2. For each trend: impact projection at 1-year horizon
  3. For each trend: impact projection at 3-year horizon
  4. For each trend: impact projection at 5-year horizon
  5. For each trend: impact projection at 10-year horizon
- **Color mapping:** Cells use intensity gradient (lighter = weaker impact, darker = stronger impact); timeline columns may use left-to-right progression coloring

### Trend-on-Trend Cross-Impact
- **Category:** Trend Analysis
- **Diagram type:** Cross-Impact Matrix
- **Tool:** XML
- **Layout:** 5×5 matrix; both rows and columns labeled with the same 5 trends. Diagonal cells (trend with itself) are neutral or marked as N/A. Off-diagonal cells show interaction effects (synergistic, neutral, antagonistic, or multiplier effect).
- **User inquiry fields:**
  1. 3–5 key trends to analyze
  2. For each pair of trends: interaction effect description (how does Trend A affect Trend B? Is it reinforcing, neutral, or inhibiting?)
  3. Strength of interaction (weak, moderate, strong, or multiplicative)
  4. Optional: examples or evidence for each interaction
- **Color mapping:** Interactions use directional coloring (green = synergistic/amplifying, yellow = neutral, red = antagonistic/inhibiting)

### Countertrends & Cycles
- **Category:** Trend Analysis
- **Diagram type:** Comparison Table
- **Tool:** XML
- **Layout:** Rows represent individual trends (1–5); columns are: Countertrends (opposing or balancing trends), Cycles (cyclical patterns or oscillations the trend experiences).
- **User inquiry fields:**
  1. 3–5 identified primary trends
  2. For each trend: identified countertrends (trends that oppose, balance, or limit it)
  3. For each trend: cyclical or oscillating patterns (if applicable, describe frequency and amplitude)
  4. Optional: evidence or examples for each countertrend and cycle
- **Color mapping:** Countertrends use contrasting color to primary trends (if trend is red, countertrend is blue); cycle indicators use wave or cyclic visual patterns

### Threat/Opportunity Analysis
- **Category:** Trend Analysis
- **Diagram type:** Comparison Table
- **Tool:** XML
- **Layout:** Rows represent individual trends (1–5). Left section: Columns = Threats (negative implications), Policy Actions (responses to threats). Right section: Columns = Opportunities (positive implications), Policy Actions (responses to opportunities).
- **User inquiry fields:**
  1. 3–5 identified trends
  2. For each trend: identified threats (negative consequences or risks)
  3. For each threat: proposed policy action or mitigation strategy
  4. For each trend: identified opportunities (positive consequences or benefits)
  5. For each opportunity: proposed policy action or strategic response
- **Color mapping:** Threats section background is light red/pink; Opportunities section is light green. Severity of threat/opportunity may use deeper color intensity.

## Category 3: Weak Signals & Perception

### Perception Analysis (See/Hear/Think/Feel)
- **Category:** Weak Signals & Perception
- **Diagram type:** Hub-and-Spoke
- **Tool:** XML
- **Layout:** Central node represents the topic being analyzed. Four main spokes radiate outward: See (visual/observable signals), Hear (reported/communicated signals), Think (analytical/inference-based observations), Feel (emotional/subjective impressions). Each spoke branches into multiple sub-nodes for specific observations.
- **User inquiry fields:**
  1. Topic or issue being analyzed (central hub)
  2. What do you see? (observable facts, visible trends, concrete data)
  3. What do you hear? (reported information, expert opinions, stakeholder statements)
  4. What do you think? (interpretations, analytical conclusions, logical inferences)
  5. What do you feel? (intuitive sense, emotional responses, gut reactions about the topic)
- **Color mapping:** Each spoke uses distinct color (See = blue, Hear = yellow, Think = purple, Feel = pink/red); intensity increases toward periphery for more detailed observations

### What/So What/Now What
- **Category:** Weak Signals & Perception
- **Diagram type:** Process Flow
- **Tool:** Mermaid
- **Layout:** Three-phase left-to-right sequential flow. Phase 1 (What) = observations and data; Phase 2 (So What) = implications and meaning; Phase 3 (Now What) = recommended actions and responses. Each phase connected by arrows showing logical progression.
- **User inquiry fields:**
  1. List of observations or findings (What—the raw signals/data)
  2. Implications of those observations (So What—what do they mean?)
  3. Recommended actions based on implications (Now What—what should be done?)
  4. Priority or urgency for each action (optional: high/medium/low)
- **Color mapping:** Phase 1 (What) = blue, Phase 2 (So What) = yellow, Phase 3 (Now What) = green; arrows show flow direction

### Weak Signal Analysis I
- **Category:** Weak Signals & Perception
- **Diagram type:** Comparison Table
- **Tool:** XML
- **Layout:** Rows represent issues or domains being monitored (1–5); columns are: Weak Signals Detected (faint signs already observed), Weak Signals Anticipated (early warning indicators to watch for), How to Monitor (methods, sources, or metrics for tracking).
- **User inquiry fields:**
  1. 3–5 issues or strategic domains being monitored
  2. For each issue: weak signals already detected (emerging indicators, faint trends, early warnings observed)
  3. For each issue: weak signals anticipated (early warning signs to expect, signals that would indicate escalation)
  4. For each issue: monitoring methods (how to detect signals—data sources, metrics, expert networks, scanning channels)
- **Color mapping:** "Detected" column uses slightly darker shade (signals already visible); "Anticipated" column lighter shade (early warning); "Monitoring" column neutral or distinct color showing methods/tools

### Weak Signal Analysis II
- **Category:** Weak Signals & Perception
- **Diagram type:** Comparison Table
- **Tool:** XML
- **Layout:** Rows represent specific weak signals detected (1–5); columns are: Signal (description of the weak signal), Actors (who is involved or driving this signal), Hypotheses (what it might mean or lead to), Questions (investigative questions to clarify understanding).
- **User inquiry fields:**
  1. 3–5 weak signals to investigate
  2. For each signal: clear description of what was observed
  3. For each signal: relevant actors or stakeholders involved
  4. For each signal: hypotheses about meaning (what could this signal indicate?)
  5. For each signal: research questions (what needs to be learned to validate or understand the signal better?)
- **Color mapping:** Rows alternate between light gray and white for readability; hypothesis and question columns use distinct background to highlight interpretive content

## Category 4: Scenario Planning (Basic)

### Three Futures
- **Category:** Scenario Planning (Basic)
- **Diagram type:** Three-Column Comparison
- **Tool:** XML
- **Layout:** Three side-by-side columns: Best Case (left), Likely Case (center), Worst Case (right). Each column contains scenario name/title and narrative summary. Color coding: green background for best case, amber/yellow for likely case, red for worst case.
- **User inquiry fields:**
  1. Strategic question or issue being explored
  2. Best Case scenario: narrative description of the most positive plausible future
  3. Likely Case scenario: narrative description of the most probable future based on current trends
  4. Worst Case scenario: narrative description of the most negative plausible future
  5. Time horizon for the scenarios (e.g., 5 years, 10 years)
- **Color mapping:** Best Case = green (positive), Likely Case = amber/yellow (neutral), Worst Case = red (negative). Text should be readable against background colors.

### Futures Cone
- **Category:** Scenario Planning (Basic)
- **Diagram type:** Concentric Arcs / Cone
- **Tool:** XML
- **Layout:** Cone expanding from left (present) to right (future). Three main bands: Probable (narrow, center—high-confidence futures), Plausible (medium width—possible futures), Possible (widest outer band—extreme but conceivable futures). Preferable future shown as dotted overlay zone within the cone. Wildcards plotted as scattered markers outside or at edges of the cone.
- **User inquiry fields:**
  1. Time horizon (e.g., 5, 10, or 20 years into the future)
  2. Description of the probable future (most likely path forward based on trends)
  3. Plausible alternatives (2–3 alternative scenarios that could reasonably occur)
  4. Possible extremes (1–2 extreme or low-probability but high-impact futures)
  5. Preferable future (the desired or optimal outcome)
  6. Wildcard events (1–3 low-probability, high-impact events that could emerge)
- **Color mapping:** Probable band = green (high confidence), Plausible band = yellow (medium confidence), Possible band = light gray (low confidence). Preferable future = dotted blue overlay. Wildcards = red or orange markers outside bands.

### Dator's Four Archetypes
- **Category:** Scenario Planning (Basic)
- **Diagram type:** Quadrant/2×2 Matrix
- **Tool:** XML
- **Layout:** Four quadrants with axes representing key uncertainty dimensions relevant to the strategic issue. Quadrants labeled: Continuation (top-right—familiar future), Transformation (top-left—radically different positive), Discipline & Constraint (bottom-left—restricted/rationed future), Collapse (bottom-right—breakdown/crisis). Each quadrant contains a detailed scenario narrative and key characteristics.
- **User inquiry fields:**
  1. Strategic issue or question being explored
  2. Two key uncertainty axes defining the quadrants (e.g., "Technology Adoption" vs. "Governance Model")
  3. Continuation narrative: a future where current trends continue (status quo evolution)
  4. Transformation narrative: a future of positive radical change or breakthrough
  5. Discipline & Constraint narrative: a future of restrictions, rationing, or imposed limits
  6. Collapse narrative: a future involving breakdown, crisis, or system failure
- **Color mapping:** Continuation = green (positive trend continuation), Transformation = blue (positive disruption), Discipline & Constraint = yellow (cautious/constrained), Collapse = red (negative/crisis)

## Category 5: Scenario Planning (Advanced)

### Intuitive Logics
- **Category:** Scenario Planning (Advanced)
- **Diagram type:** Quadrant Matrix + Comparison Table
- **Tool:** XML
- **Layout:** Two integrated diagrams. Diagram 1: 2×2 matrix with two key uncertainty axes creating four scenario cells (A, B, C, D), each with scenario name. Diagram 2: Comparison table with rows = Risks, Opportunities, Strategy, Resources, Knowledge; columns = Scenario A, B, C, D. Each cell contains relevant assessment for that scenario.
- **User inquiry fields:**
  1. Two key uncertainties (the axes that define the scenarios—e.g., "Economic Growth" and "Technology Disruption")
  2. Scenario name for each quadrant (A, B, C, D)
  3. For each scenario: identified risks (potential negative outcomes or challenges)
  4. For each scenario: identified opportunities (potential benefits or advantages)
  5. For each scenario: recommended strategy or approach
  6. For each scenario: resources needed (budget, skills, capabilities)
  7. For each scenario: knowledge required (what needs to be learned or understood)
- **Color mapping:** Each scenario uses a distinct color in the matrix. Comparison table: Risks = light red, Opportunities = light green, Strategy = light blue, Resources = light yellow, Knowledge = light purple.

### Lum's Four Steps to the Future
- **Category:** Scenario Planning (Advanced)
- **Diagram type:** Sequential Process Flow
- **Tool:** Mermaid
- **Layout:** Four-step left-to-right sequential flow: Step 1 = Understanding the Past → Step 2 = Analyzing the Present → Step 3 = Forecasting the Future → Step 4 = Envisioning the Aspiration. Each step is a phase box with sub-nodes or bullet points containing key findings, patterns, or outputs from that step.
- **User inquiry fields:**
  1. Step 1 (Understanding the Past): key historical patterns, precedents, or historical lessons relevant to the issue
  2. Step 2 (Analyzing the Present): current conditions, trends, drivers, and constraints
  3. Step 3 (Forecasting the Future): forecasted scenarios, projected trends, anticipated disruptions
  4. Step 4 (Envisioning the Aspiration): desired future state, aspirational goals, values to uphold
  5. Optional: connections or dependencies between steps
- **Color mapping:** Each step uses a distinct color (Step 1 = blue/past, Step 2 = yellow/present, Step 3 = orange/near future, Step 4 = green/aspiration). Arrows between steps use neutral gray.

### Layered Timeline (Lum)
- **Category:** Scenario Planning (Advanced)
- **Diagram type:** Multi-Row Swimlane
- **Tool:** XML
- **Layout:** Three horizontal swimlanes representing analytical layers (Daily Life, Systems, Values/Worldviews) arranged vertically. Horizontal time axis (Past → Present → Future) divided into time periods. Cells within swimlanes contain events, observations, or patterns relevant to that layer and time period.
- **User inquiry fields:**
  1. Time period and specific dates/years being analyzed (e.g., 1995–2025, or specific decades)
  2. Daily Life layer—Past: observations about daily routines, behaviors, consumer habits in the past
  3. Daily Life layer—Present: current daily patterns and behaviors
  4. Daily Life layer—Future: projected changes in daily life
  5. Systems layer—Past: historical institutional, technological, or organizational systems
  6. Systems layer—Present: current systems and infrastructure
  7. Systems layer—Future: anticipated system changes or transformations
  8. Values/Worldviews layer—Past: prevailing beliefs, values, and worldviews in the past
  9. Values/Worldviews layer—Present: current values and worldviews
  10. Values/Worldviews layer—Future: projected value shifts or worldview changes
- **Color mapping:** Each layer uses a distinct color (Daily Life = light blue, Systems = light green, Values/Worldviews = light purple). Time periods may use left-to-right gradient (past = pale, future = darker).

## Category 6: Assessment Frameworks

### VUCA Analysis
- **Category:** Assessment Frameworks
- **Diagram type:** Scored Assessment Matrix
- **Tool:** XML
- **Layout:** Two integrated diagrams. Diagram 1: 4 rows (Volatility, Uncertainty, Complexity, Ambiguity) × 4 columns (Local, National, Regional/Europe, Global). Each cell contains a severity rating (e.g., Low/Medium/High or numeric 1–5 scale). Diagram 2: VUCA Responses table with rows = Element (V/U/C/A), columns = Knowledge, Investments, Actions, Perspectives. Each cell contains strategic response.
- **User inquiry fields:**
  1. Geographic scopes to assess (Local, National, Regional/Europe, Global)
  2. For each VUCA element and geographic scope: severity rating (Low, Medium, High, or 1–5 scale)
  3. For Volatility: knowledge needed, investments required, actions to take, perspective shifts required
  4. For Uncertainty: knowledge needed, investments required, actions to take, perspective shifts required
  5. For Complexity: knowledge needed, investments required, actions to take, perspective shifts required
  6. For Ambiguity: knowledge needed, investments required, actions to take, perspective shifts required
- **Color mapping:** Assessment matrix uses intensity gradient (light = low severity, dark = high severity). Responses table uses color per VUCA element (Volatility = red, Uncertainty = orange, Complexity = blue, Ambiguity = purple).

### CATUR Analysis
- **Category:** Assessment Frameworks
- **Diagram type:** Scored Assessment Matrix
- **Tool:** XML
- **Layout:** 5 rows (Complexity, Ambiguity, Tension, Uncertainty, Risk) × 4 columns (Local, National, Regional/Europe, Global). Each cell contains a severity rating (e.g., Low/Medium/High or numeric 1–5 scale).
- **User inquiry fields:**
  1. Geographic scopes to assess (Local, National, Regional/Europe, Global)
  2. For each CATUR element and geographic scope: severity rating (Low, Medium, High, or 1–5 scale)
  3. Optional: brief description or justification for each rating
  4. Priority areas for focus (which combinations of element and geography are most critical?)
- **Color mapping:** Intensity gradient based on severity rating (light = low severity, dark = high severity). Individual CATUR elements may use distinct colors or uniform gradient across all.

### What If/Then Cross-Impact
- **Category:** Assessment Frameworks
- **Diagram type:** Cross-Impact Matrix
- **Tool:** XML
- **Layout:** Two integrated tables. Table 1: Columns = What If...? (hypothetical scenario), Triggers (what would cause it), Indicators (how would we detect it), Event Narrative (detailed description), Impact (consequences). Table 2: Columns = If (Action), Then (Reaction/consequence), Risks of Action, Risks of Inaction. Rows populated for 3–5 what-if scenarios or action-reaction pairs.
- **User inquiry fields:**
  1. 3–5 "What If" hypothetical scenarios
  2. For each scenario: trigger conditions (what would cause this scenario to occur?)
  3. For each scenario: early indicators or warning signs
  4. For each scenario: narrative description of the scenario unfolding
  5. For each scenario: anticipated impacts (consequences, ripple effects)
  6. 3–5 policy actions or strategic decisions
  7. For each action: anticipated reactions or consequences
  8. For each action: risks of taking the action
  9. For each action: risks of not taking the action
- **Color mapping:** Table 1 uses neutral gray; Table 2 uses red background for "Risks of Action" and orange for "Risks of Inaction" to highlight consequences.

### Consequence Analysis
- **Category:** Assessment Frameworks
- **Diagram type:** Comparison Table
- **Tool:** XML
- **Layout:** Rows represent trend/trigger events (1–5); columns are organized in three narrative scenarios (Narrative 1, 2, 3), each with sub-rows for Positive Impacts, Negative Impacts, Policy Options, and Planning Considerations. Alternatively, a single large matrix with rows = Trigger, Narrative 1–3 + sub-rows for impacts and responses.
- **User inquiry fields:**
  1. 3–5 trend events or trigger scenarios to analyze
  2. For each trigger event: three alternative narrative trajectories (how the event could unfold under different conditions)
  3. For each narrative: positive impacts (benefits, opportunities)
  4. For each narrative: negative impacts (risks, challenges, costs)
  5. For each narrative: policy options or strategic responses
  6. For each narrative: planning considerations (preparation needed, resources required)
- **Color mapping:** Positive impacts = light green, Negative impacts = light red, Policy options = light blue, Planning = light yellow. Each narrative may use a distinct column color or alternating row shading.

## Category 7: Comparative & Historical

### Analogical Reasoning
- **Category:** Comparative & Historical
- **Diagram type:** Two-Column Comparison
- **Tool:** XML
- **Layout:** Two main comparison tables. Table 1: Columns = Current Issue Elements, Historical Parallel Elements, Similarities (how they align). Table 2: Columns = Current Issue Elements, Historical Parallel Elements, Differences (how they diverge). Below the tables: seven structured lesson questions with response fields: (1) What worked then? (2) Why did it work? (3) What didn't work? (4) Why didn't it work? (5) What's different now? (6) What similar mistakes might we make? (7) What lessons apply today?
- **User inquiry fields:**
  1. Current issue or challenge being analyzed
  2. Historical parallel or precedent (what past event/situation is analogous?)
  3. Similarities between current issue and historical parallel (specific elements that match)
  4. Differences between current issue and historical parallel (specific elements that diverge)
  5. Response to each of the seven lesson questions
- **Color mapping:** Table 1 (Similarities) uses light green background; Table 2 (Differences) uses light orange background. Lesson questions use neutral white/light gray with emphasis on responses.

### Stakeholder Analysis (Lum)
- **Category:** Comparative & Historical
- **Diagram type:** Hub-and-Spoke or Comparison Table
- **Tool:** XML
- **Layout:** Can be rendered as either: (1) Hub-and-Spoke with central issue and four spokes representing stakeholder categories (Resisters, Channelers, Accelerators, Emerging), or (2) Comparison Table with rows = stakeholder names/groups, columns = stakeholder category and four disruption scenarios (Continuity, Low Disruption, High Disruption, Abrupt Change). Each cell identifies stakeholder stance or strategy in that scenario.
- **User inquiry fields:**
  1. Strategic issue or change initiative being analyzed
  2. Resisters: stakeholders who oppose or slow the change (names, groups, organizations)
  3. Channelers: stakeholders who manage or regulate change (names, groups, organizations)
  4. Accelerators: stakeholders who drive or promote change (names, groups, organizations)
  5. Emerging: new or rising stakeholders who will influence change (names, groups, organizations)
  6. For each stakeholder group: stance or behavior under Continuity scenario (status quo)
  7. For each stakeholder group: stance or behavior under Low Disruption scenario
  8. For each stakeholder group: stance or behavior under High Disruption scenario
  9. For each stakeholder group: stance or behavior under Abrupt Change scenario
- **Color mapping:** Resisters = red, Channelers = yellow, Accelerators = green, Emerging = blue. Disruption scenarios may use intensity gradient (Continuity = light, Abrupt Change = dark) or be arranged left-to-right in increasing disruption.

## Implementation Notes

### Universal User Inquiry Principles
- Always ask for clear, specific input rather than vague statements
- Request at least 3 examples or data points per field
- Specify quantity (e.g., "3–5 trends," "up to 10 participants")
- Ask for time horizons or date ranges when relevant
- Request severity ratings on clear scales (Low/Medium/High, or 1–5)
- Encourage use of concrete examples or evidence

### Diagram Creation Process
1. Collect user inquiry fields in order of listing
2. Validate completeness of input before diagram generation
3. Apply color mapping consistently within a technique
4. Use consistent labeling conventions across all techniques
5. Ensure accessibility: readable font sizes, sufficient color contrast, text alternatives for visual elements

### Tool Selection Guidelines
- **XML:** Complex layouts, matrices, cross-impacts, multi-panel diagrams
- **Mermaid:** Sequential flows, process diagrams, simple hierarchies
- **CSV:** Large, highly structured data tables (rare in this catalog; typically converted to XML for visualization)

### Technique Sequencing for Users
- Start with **Environmental Scanning** (Category 1) to understand the landscape
- Progress to **Trend Analysis** (Category 2) to identify key drivers
- Use **Weak Signals** (Category 3) for early warning and interpretation
- Develop scenarios via **Scenario Planning** (Categories 4–5) for exploring futures
- Evaluate scenarios using **Assessment Frameworks** (Category 6)
- Learn from **Comparative & Historical** (Category 7) analysis to inform strategy
